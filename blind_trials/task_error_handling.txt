# Task: Implement a small batch pipeline with retry classification and cleanup

Create a file called `subject_solution.nim`.

The program must be small enough to judge mechanically, but open enough that boundary placement and error-handling choices still matter.

Follow this task exactly. If a requirement below says a proc, type, field, or constant must exist, keep that name and public/export marker exactly as written.

## Required public constants and types

```nim
const
  ExitAllOk* = 0
  ExitPartialFailure* = 1
  ExitFatalRuntime* = 2

type
  Positive* = range[1 .. high(int)]

  ItemErrorKind* = enum
    NoError,
    InputError,
    NetworkError,
    RateLimit,
    Timeout,
    ParseError,
    HttpError

  ItemOutcome* = object
    id*: string
    attempts*: int
    success*: bool
    value*: int
    errorKind*: ItemErrorKind
    errorMsg*: string

  BatchSummary* = object
    okCount*: int
    failCount*: int
    outcomes*: seq[ItemOutcome]
```

Do not add any other public result-like type for intermediate steps.

## Required fake helpers

Include these helpers exactly or with equivalent behavior:

```nim
type
  FakeReply = object
    transportError: string
    statusCode: int
    body: string

proc fakeLoadInput(id: string): string =
  if id.len == 0:
    raise newException(IOError, "input id empty")
  case id
  of "missing":
    raise newException(IOError, "input missing")
  of "blank":
    result = ""
  else:
    result = "doc:" & id

proc fakeSend(payload: string; attempt: int): FakeReply =
  case payload
  of "doc:retry-ok":
    if attempt == 1:
      FakeReply(transportError: "", statusCode: 429, body: "")
    else:
      FakeReply(transportError: "", statusCode: 200, body: "value=21")
  of "doc:transport-retry-ok":
    if attempt == 1:
      FakeReply(transportError: "timeout", statusCode: 0, body: "")
    else:
      FakeReply(transportError: "", statusCode: 200, body: "value=34")
  of "doc:parse-bad":
    FakeReply(transportError: "", statusCode: 200, body: "not-an-int")
  of "doc:http-bad":
    FakeReply(transportError: "", statusCode: 400, body: "")
  else:
    FakeReply(transportError: "", statusCode: 200, body: "value=" & $payload.len)

proc fakeAuditWrite(auditPath: string; line: string) =
  if auditPath == "audit-fail":
    raise newException(OSError, "audit write failed")

var closeCount = 0
var abortCount = 0

type
  FakeClient = ref object

proc newFakeClient(): FakeClient =
  FakeClient()

proc closeClient(client: FakeClient) =
  inc closeCount

proc abortClient(client: FakeClient) =
  inc abortCount

proc resetClientCounters*() =
  closeCount = 0
  abortCount = 0

proc clientCounters*(): tuple[closed: int, aborted: int] =
  (closeCount, abortCount)
```

## Required public procs

1. `parseRetryLimit*(s: string; value: var Positive): bool`
   - Parse a retry limit.
   - Return `true` on success.
   - Return `false` on invalid input.

2. `shouldRetry*(transportError: string; statusCode, attempt: int; retryLimit: Positive): bool`
   - Return `true` only when the current failure is retryable and `attempt < retryLimit`.
   - Retry transport errors `"timeout"` and `"disconnect"`.
   - Retry HTTP status `429`, `408`, and `504`.
   - Do not retry any other transport error or HTTP status.

3. `classifyFinalFailure*(transportError: string; statusCode: int): ItemErrorKind`
   - If `transportError == "timeout"`, return `Timeout`.
   - Else if `transportError.len > 0`, return `NetworkError`.
   - Else if `statusCode == 429`, return `RateLimit`.
   - Else if `statusCode == 408 or statusCode == 504`, return `Timeout`.
   - Else return `HttpError`.

4. `runBatch*(ids: seq[string]; retryLimit: Positive; auditPath: string): BatchSummary`
   Process items in input order and return one `ItemOutcome` per input item, unless audit logging fails fatally.

   For each item:
   - Load input with `fakeLoadInput`.
   - If the loaded payload is empty, record an `InputError` outcome. Do not call `fakeSend`.
   - Otherwise call `fakeSend`.
   - If `shouldRetry(...)` says retry, retry until success or final failure.
   - On HTTP `200`, parse only bodies of the form `value=<int>`.
   - If body parsing fails, record `ParseError`.
   - On final non-success transport or HTTP response, record `classifyFinalFailure(...)`.

   Outcome rules:
   - Success outcome:
     - `success = true`
     - `value > 0`
     - `errorKind = NoError`
     - `errorMsg.len == 0`
   - Failure outcome:
     - `success = false`
     - `value = 0`
     - `errorKind != NoError`
     - `errorMsg.len > 0`

   Attempt counting rules:
   - `attempts` must be the number of calls made to `fakeSend` for that item.
   - Input failures that never call `fakeSend` must still record `attempts = 1`.

   Audit rules:
   - On every failed item, call `fakeAuditWrite(auditPath, line)`.
   - If `fakeAuditWrite` raises `OSError`, catch it, add local context, and re-raise as `IOError`.
   - If audit logging fails, stop the batch immediately and let that exception escape.

5. `runApp*(ids: seq[string]; retryLimit: Positive; auditPath: string): int`
   - Create a `FakeClient` with `newFakeClient()`.
   - Call `runBatch(...)`.
   - Return:
     - `ExitAllOk` if every item succeeded
     - `ExitPartialFailure` if the batch completed and at least one item failed
     - `ExitFatalRuntime` if a fatal exception escaped
   - Cleanup rules:
     - On non-fatal completion, call `closeClient`.
     - On fatal exception, call `abortClient`.
     - Always do cleanup exactly once.

## Required smoke run

Add a `when isMainModule:` block that performs exactly these checks:

1. Parse helper:
   - one valid parse, one invalid parse

2. Retry helper:
   - `shouldRetry("", 429, 1, 2) == true`
   - `shouldRetry("", 429, 2, 2) == false`
   - `shouldRetry("timeout", 0, 1, 2) == true`
   - `shouldRetry("", 400, 1, 2) == false`

3. Final-failure classifier:
   - `classifyFinalFailure("timeout", 0) == Timeout`
   - `classifyFinalFailure("disconnect", 0) == NetworkError`
   - `classifyFinalFailure("", 429) == RateLimit`
   - `classifyFinalFailure("", 400) == HttpError`

4. Batch run:

```nim
let summary = runBatch(@[
  "good",
  "retry-ok",
  "transport-retry-ok",
  "parse-bad",
  "missing",
  "blank",
  "http-bad"
], 2, "audit.log")
```

The smoke run must assert all of these:
- `summary.okCount == 3`
- `summary.failCount == 4`
- `summary.outcomes.len == 7`
- outcomes stay in the same order as the input ids
- `"retry-ok"` succeeds with `attempts == 2`
- `"transport-retry-ok"` succeeds with `attempts == 2`
- `"parse-bad"` fails with `errorKind == ParseError`
- `"missing"` fails with `errorKind == InputError`
- `"blank"` fails with `errorKind == InputError`
- `"http-bad"` fails with `errorKind == HttpError`

5. App cleanup:
   - after `resetClientCounters()`, `runApp(@["good", "retry-ok"], 2, "audit.log") == ExitAllOk`
   - counters must then be `closed = 1`, `aborted = 0`
   - after `resetClientCounters()`, `runApp(@["missing"], 2, "audit-fail") == ExitFatalRuntime`
   - counters must then be `closed = 0`, `aborted = 1`

6. Print:

```nim
echo "SMOKE: PASS"
```

## Critical requirements

- Use `CatchableError`, not bare `Exception`
- Do not add manual `retryLimit <= 0` checks; `retryLimit` is already `Positive`
- Do not introduce ad-hoc intermediate result objects such as `StepResult`, `OkOrError`, or `ResultInfo`
- Do not wrap every internal call in its own local `try/except`
- Do not catch and re-raise the same exception type unless you add real local context
- Use descriptive public names; avoid generic names such as `Result`, `Data`, or `handleError`
- Compile and run with `nim c -r --mm:orc`
