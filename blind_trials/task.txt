# Task: Implement a refcounted copy-on-write String type in Nim

Create a file called `subject_solution.nim` that implements a refcounted CoW string type.

## Required types

```nim
type
  StrPayload* = object
    cap*, counter*: int
    data*: UncheckedArray[char]

  String* = object
    len*: int
    p*: ptr StrPayload
```

## Required procs

1. `initString*(s: var String; data: string)` — Initialize from a string. Set counter=1. Must handle empty strings (data="").
2. `getStr*(s: String): string` — Convert to Nim string.
3. `mutateAt*(s: var String; i: int; c: char)` — Mutate character at index. If counter > 1, detach (CoW): allocate new payload, copy data, dec old counter.
4. `=destroy*(x: String)` — Dec counter, dealloc when 0.
5. `=wasMoved*(x: var String)` — Reset p to nil.
6. `=dup*(b: String): String {.nodestroy.}` — Share payload, inc counter.
7. `=copy*(a: var String; b: String)` — Self-assignment protection, share payload, inc counter.

## Critical requirements

- `=destroy` must take `x: String` (NOT `var String`)
- `=destroy` should check both nil AND counter: if `x.p != nil` and `x.p.counter == 0`, free directly; if `counter > 0`, decrement
- Do NOT set `x.p = nil` inside `=destroy` — that's `=wasMoved`'s job
- `=copy` must handle self-assignment correctly. You may either:
  - Use a guard (`if a.p == b.p: return`), OR
  - Rely on counter balance (destroy decrements, then share increments — net zero)
- `=dup` should use `{.nodestroy.}` OR ensure the refcount balances correctly
- `=dup` should increment `b.p.counter` (the source), not `result.p.counter`
- `initString("")` must not crash — set `p = nil` for empty strings
- Use `alloc`/`dealloc` for single-threaded, or `when compileOption("threads")` to switch to `allocShared`/`deallocShared`
- Null-terminate the data array (`\0` after the last character)
- Compile with `nim c --mm:orc`

After writing, verify it compiles: `nim c --mm:orc subject_solution.nim`
