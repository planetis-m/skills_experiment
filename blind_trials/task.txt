# Task: Implement a refcounted copy-on-write String type in Nim

Create a file called `subject_solution.nim` that implements a refcounted CoW string type.

Use the inverted counter convention throughout this task:
- `counter == 0` means the payload is uniquely owned
- `counter > 0` means the payload is shared

## Required types

```nim
type
  StrPayload* = object
    cap*, counter*: int
    data*: UncheckedArray[char]

  String* = object
    len*: int
    p*: ptr StrPayload
```

## Required procs

1. `initString*(s: var String; data: string)` — Initialize from a string. Set `counter = 0` for non-empty payloads. Must handle empty strings (data = "") by keeping `p = nil`.
2. `getStr*(s: String): string` — Convert to Nim string.
3. `mutateAt*(s: var String; i: int; c: char)` — Mutate character at index. If `counter > 0`, detach: allocate a new unique payload, copy data, decrement the old payload, then mutate.
4. `=destroy*(x: String)` — If `x.p != nil` and `counter == 0`, free directly. If `counter > 0`, decrement.
5. `=wasMoved*(x: var String)` — Reset p to nil.
6. `=dup*(b: String): String` — Share payload and increment the counter.
7. `=copy*(a: var String; b: String)` — Destroy the old payload, then share the new one by incrementing the counter.

## Critical requirements

- `=destroy` must take `x: String` (NOT `var String`)
- Do NOT set `x.p = nil` inside `=destroy` — that's `=wasMoved`'s job
- `=copy` and `=dup` must preserve the inverted counter arithmetic
- Alias assignment on the same payload must remain balanced
- `=dup` should increment `b.p.counter` (the source), not `result.p.counter`
- `initString("")` must not crash — set `p = nil` for empty strings
- Use `alloc`/`dealloc` for single-threaded, or `when compileOption("threads")` to switch to `allocShared`/`deallocShared`
- Null-terminate the data array (`\0` after the last character)
- Compile with `nim c --mm:orc`

After writing, verify it compiles: `nim c --mm:orc subject_solution.nim`
