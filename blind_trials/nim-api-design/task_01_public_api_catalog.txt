# Task: Implement a small Nim library with one coherent public API

Create a file called `subject_solution.nim`.

The goal is to judge API design, not algorithm difficulty. The runtime behavior is fixed. Most API-shape decisions are yours.

## Required public surface map

Near the top of `subject_solution.nim`, add this comment block and replace the names on the right-hand side with the exported names you chose:

```nim
# API surface:
# catalog type: PackageCatalog
# empty constructor: initPackageCatalog
# seed constructor: toPackageCatalog
# length query: len
# membership query: contains
# insert or replace: put
# primary read entrypoint: meta
# tag mutation entrypoint: tags
# convenience readers: none
```

For `convenience readers`, either write `none` or a comma-separated list such as `versionOf, downloadsOf`.

## Required public types

These exact public types must exist:

```nim
type
  PackageId* = distinct string

  PackageMeta* = object
    title*: string
    version*: string
    tags*: seq[string]
    downloads*: Natural
```

Also export exactly one primary public catalog/container type. Its name is your choice. List it in the API surface map.

`PackageId` must expose these exact borrowed operations:

```nim
proc `==`*(a, b: PackageId): bool {.borrow.}
proc `$`*(id: PackageId): string {.borrow.}
```

## Required API capabilities

Your public API must let a caller do all of these:

- construct an empty catalog
- construct a catalog from `openArray[(PackageId, PackageMeta)]`
- query the catalog length
- check membership by `PackageId`
- insert a new package
- replace an existing package
- read metadata for a package through one primary public read entrypoint
- mutate stored tags through one clear public tag-mutation path
- observe replacement through the public read surface
- get a specific catchable exception when a required package is missing

You may add convenience readers such as `versionOf` or `downloadsOf`, but they are optional.

## Critical design requirements

- Do not add any public status tuple such as `(ok: bool, value: T, errorMsg: string)`.
- Do not add a public `Ref`, `Handle`, or second public catalog representation.
- Do not silently return default values for missing packages.
- Do not export scalar `var` accessors such as `var int`, `var Natural`, or `var string`.
- If you use `lent` or `var` accessors that borrow from catalog storage, return directly from storage. Do not route through a temp local.
- Use descriptive public names. Avoid generic names such as `Result`, `Data`, or `handleError`.
- Compile and run with `nim c -r --mm:orc`.

Whether an internal helper is exported is not scored unless it creates a second public API path.

## Required smoke run

Add a `when isMainModule:` block that builds this seed data:

```nim
let core = PackageId("core")
let cli = PackageId("cli")

let seed = [
  (core, PackageMeta(title: "Core", version: "1.0.0", tags: @["runtime"], downloads: 10)),
  (cli, PackageMeta(title: "CLI", version: "1.1.0", tags: @["tooling"], downloads: 5))
]
```

Then use your chosen public API to assert all of these:

- a catalog constructed from `seed` has length `2`
- the catalog contains `core`
- the catalog does not contain `PackageId("missing")`
- the primary read entrypoint exposes title `"Core"` for `core`
- the public tag-mutation path adds one tag to `cli`
- the stored tags for `cli` then have length `2`
- replacing `core` with version `"2.0.0"` and downloads `11` is reflected by the public read surface
- missing-package reads raise `KeyError` or `ValueError`

Then print:

```nim
echo "SMOKE: PASS"
```

## Judge checklist

Score only these checks:

- compiles and runs with `nim c -r --mm:orc subject_solution.nim`
- runtime prints `SMOKE: PASS`
- `PackageId` is a `distinct string` with borrowed `==` and `$`
- exactly one primary public catalog type exists
- the public API exposes one empty constructor and one seed constructor
- there is no second public catalog representation
- public semantic data uses named objects, not status tuples
- the read surface is coherent: one primary metadata read entrypoint, plus only optional convenience readers
- the mutation surface is narrow: one clear tag-mutation path and no scalar `var` accessors
- the API surface map is present and matches the exported API
- missing data is not reported via silent defaults
- missing-package reads raise a specific catchable exception
- borrowed accessors, if used, do not rely on temp locals that create escaping-borrow issues
- public names are descriptive rather than generic

After writing, verify it runs:

```bash
nim c -r --mm:orc subject_solution.nim
```
